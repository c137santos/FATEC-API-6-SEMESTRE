## 🎯 1. Overview

This feature combines ANEEL regulatory indicators with the network’s geographic database to generate a heat map, mathematically demonstrating which concession areas require investment.

* **Objective:** Allow the technical/commercial team to generate and visualize a heat map of the utility company’s electrical groups based on deviations from DEC and FEC targets.
* **Problem:** The commercial team struggles to visually and convincingly demonstrate the network issues to utility engineers. Without spatial data, the sales argument for smart sensors loses technical credibility.
* **KPI / Success:** Electrical groups accurately mapped and the heat map successfully rendered.

## 📋 3. Narrative

**As** a Tecsys commercial/technical consultant,
**I want** to visualize a georeferenced heat map that colors the network segments according to the most critical electrical circuits based on their Criticality Index (Green, Orange, or Red),
**So that** I can demonstrate to the Electrical Engineer, using official regulatory data, which lines are the most critical and justify investment in smart sensors.

---

### Happy Path

1. The system consumes the previously calculated criticality data.
2. The system consumes geographic data (GeoJSON or Shapefile).
3. The system matches the criticality data with the electrical groups.
4. The system renders the georeferenced map.
5. The system applies colors to the groups according to the criticality level.
6. The user accesses the screen and visualizes the map with the classified regions.

---

## 🛡️ 4. Business Rules

* **Data Sources:** Open Data Portal - BDGD, Collective Indicators Dataset (DEC/FEC).
* **Dependency:** This User Story is technically blocked until [US 01](/docs/user-stories/user-story-01.md) (Criticality Calculation and Ranking Table) is approved, since the map will consume the data generated by it.
* **Spatial Data:** The graphical interface must read ANEEL Shapefile or GeoJSON files (BDGD database) to draw the boundaries of each electrical group before applying the colors.

## ✅ 5. Acceptance Criteria

### Geographic Mapping and Color Application

| Step           | Action|
| -------------- | ----------------- |
| **Given that** | the Criticality Index has already been calculated|
| **When**       | the map is rendered|
| **Then**       | the network segments must be filled with: 🟢 Green (Criticality = 0%), 🟠 Orange (Criticality between 0 and 10%), 🔴 Red (Criticality > 10%) |
